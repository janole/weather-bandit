# Weather Outlook

This is the default Weather Bandit GitHub Pages template. It renders
cross-validated daily outlooks as polished public pages while keeping the
source artifacts (Markdown + JSON) reviewable in git.

## Publish an outlook

```sh
# 1. Export the canonical Markdown (with frontmatter) + JSON artifact
mkdir -p outlooks/2026-07-05-berlin
weather-bandit export-md Berlin --out outlooks/2026-07-05-berlin
```

This writes `outlooks/2026-07-05-berlin/2026-07-05-berlin.md` and a matching
`.json`. For a clean permalink, rename the Markdown to `index.md`:

```sh
mv outlooks/2026-07-05-berlin/2026-07-05-berlin.md outlooks/2026-07-05-berlin/index.md
```

Review both files before committing or pushing.

## Optional: hero image

The frontmatter ships with an empty `heroImage:` slot. An agent with image
generation can produce a hero image and point the field at it:

```yaml
heroImage: /outlooks/2026-07-05-berlin/hero.jpg
heroAlt: "Clear sky over Berlin at dawn"
heroCredit: "Generated by <agent> using style: photorealistic"
```

Agents without image generation leave the slot empty — the page renders
without an image.

## GitHub Pages

Enable Pages for the repository in GitHub settings. Use the `main` branch and
the repository root as the publishing source.

The template uses Jekyll defaults in `_config.yml` so files under
`outlooks/<slug>/index.md` automatically use the `outlook` layout.

## Theming

The default Swiss/data-viz theme lives in `assets/outlook.css` and exposes its
palette and spacing as CSS custom properties on `:root`. Override them in a
downstream stylesheet (loaded after `outlook.css`) to retheme without touching
the layout markup:

```css
:root {
  --accent: #b91c1c;      /* e.g. a warm-red theme */
  --paper: #fafaf9;
  --ink: #1c1917;
}
```

## Design fixture

Open `.design/outlook-page.html` directly in a browser to test layout changes
without running Jekyll or publishing to GitHub Pages. It uses the same CSS as
the real site and includes long titles, metadata, data tables, and inline code
that should not create horizontal page scroll.
